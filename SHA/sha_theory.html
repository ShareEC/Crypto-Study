HMAC (Hash-based Message Authentication Code)

단 하나의 마스터 비밀키를 변형하여, SHA-256 믹서기를 이중(Sandwich)으로 가동하는 완벽한 방어 아키텍처

Step 1 키 분화 (Key Derivation)

사전에 약속된 1개의 마스터 비밀키에 서로 다른 패턴(ipad, opad)을 덧씌워 안쪽 키(Inner Key)와 바깥쪽 키(Outer Key) 두 개를 파생시킵니다.

Step 2 이중 샌드위치 포장

  • 1 안쪽 믹서기 가동 (Inner Hash) SHA256(Inner Key + Message) → 1차 결과물인 '중간 해시' 탄생
  • 2 바깥쪽 믹서기 가동 (Outer Hash) SHA256(Outer Key + 1차 중간 해시) → 최종 HMAC 서명 도장 완성!
길이 연장 공격 완벽 차단

해커가 중간 해시값을 훔쳐 뒤에 메시지를 이어 붙이려 해도, 마지막에 '바깥쪽 키(Outer Key)'로 한 번 더 전체를 믹서기에 돌려버리기 때문에 키를 모르는 해커는 절대 위조 서명을 만들어낼 수 없습니다.

SHS란 무엇인가?

데이터의 무결성(Integrity)을 확인하기 위해 메시지의 '압축된 표현'인 메시지 다이제스트(Message Digest)를 생성하는 표준입니다.

🔑 주요 속성

  • 충돌 회피 (Collision Resistance):
    서로 다른 입력이 같은 출력을 가질 수 없음.
  • 단방향성 (One-way):
    해시값에서 원본을 복구할 수 없음.
  • 눈사태 효과 (Avalanche Effect):
    입력의 미세한 변경이 결과를 완전히 바꿈.

(블록 크기 = $16 \times 64 = 1024$ bits)

4단계-1: 도구 정의 - 논리 함수 (Logic Tools)

본격적인 압축(Compress)을 수행하기 위해 필요한
비트 연산 도구들을 먼저 정의합니다.

Ch (Choose)

선택 함수

"x가 1이면 y를, 0이면 z를 선택합니다."

(x ∧ y) ⊕ (¬x ∧ z)

Maj (Majority)

다수결 함수

"x, y, z 중 0과 1의 개수가 더 많은 쪽을 따릅니다."

(x ∧ y) ⊕ (x ∧ z) ⊕ (y ∧ z)
Σ

Σ (Big Sigma) & σ (Small sigma)

확산(Diffusion)을 위한 뒤섞기

비트를 회전(ROTR)하고 밀어서(SHR) 위치를 섞어버립니다.
* 입력값의 작은 변화가 전체로 퍼지게 하는 핵심 역할(눈사태 효과)

For Main Loop (압축용)
Σ₀(a) = ROTR²(a) ⊕ ROTR¹³(a) ⊕ ROTR²²(a)
For Expansion (확장용)
σ₀(x) = ROTR⁷(x) ⊕ ROTR¹⁸(x) ⊕ SHR³(x)

구현 및 보안 고려사항 (Implementation Details)

안전한 시스템 구축을 위해 알고리즘 자체보다 운용 환경과 보안 모델이 중요합니다.

📜

NIST CMVP 검증

공공 및 금융 분야 도입 시 FIPS 140-3 인증을 받은 모듈 사용이 필수적입니다. 직접 구현한 코드는 검증되지 않았으므로 상용 환경 사용을 지양해야 합니다.

⏱️

부채널 공격 방지

연산 시간(Timing)이나 전력 소모 패턴을 통한 키 유출을 막기 위해 Constant-Time Implementation이 필요합니다. (조건문 분기 최소화 등)

🔐

패스워드 저장 시 주의

SHA-256은 속도가 빨라 Brute-Force 공격에 취약합니다. 패스워드 저장 시에는 PBKDF2, Argon2 등 Key Stretching 기술과 Salt를 반드시 병행해야 합니다.

🛡️

Length Extension Attack

단순 \( H(k||m) \) 구조는 길이 연장 공격에 취약합니다. 메시지 인증 코드(MAC) 구현 시에는 를 사용해야 안전합니다.
