Cryptographic Primitives

Advanced Encryption Standard

AES 암호화의 수학적 배경인 갈로아 필드부터 실제 라운드 변환 과정까지의 핵심 원리를 다룹니다.

1 핵심 정의 (Core Definitions)

AES 개요

  • 정의: 전 세계 표준 대칭키 블록 암호화 알고리즘 (SPN 구조)
  • 블록 크기: 128비트 (16바이트) 고정
  • State: 모든 연산의 기본 단위인 \(4 \times 4\) 바이트 행렬 (위 그림 참조)

키 길이(\(N_k\)) 및 라운드(\(N_r\))

  • 128 bit 10 Rounds
  • 192 bit 12 Rounds
  • 256 bit 14 Rounds

핵심 개념: AES는 단순한 비트 섞기가 아닌, GF(2^8)이라는 특수한 수학적 공간(유한체) 위에서의 연산입니다.

2 핵심 엔진: 갈로아 필드 (Galois Field \(GF(2^8)\))

특징

  • 닫힌 세계 (Closed System):
    0~255 (1 Byte)의 숫자만 사용합니다.
  • 오버플로우 방지:
    어떤 연산을 해도 결과는 항상 8비트(00~FF) 내에 존재합니다.
  • 하드웨어 최적화:
    덧셈과 뺄셈은 컴퓨터의 XOR (\(\oplus\)) 연산과 100% 동일하여 처리 속도가 매우 빠릅니다.

세계관의 헌법: 기약 다항식

Irreducible Polynomial m(x)

x^8 + x^4 + x^3 + x + 1

Hex: 0x11b

  • • 정체: 다항식 세계의 '소수(Prime)' 역할
  • • 목적: 곱셈 연산(MixColumns 등) 중 값이 8비트(255)를 넘어갈 때, 나머지 연산(Modulo)을 수행하는 기준 값.

3 비선형성의 핵심: S-Box (Substitution Box)

목적 (Confusion): AES의 유일한 비선형(Non-linear) 구조입니다. 선형 관계를 파괴하여 역추적을 불가능하게 만듭니다. 복잡한 수식을 매번 계산하지 않고, 미리 계산된 256개 값의 테이블(Look-up Table)을 참조하여 고속으로 처리합니다.

생성 과정 (Generation Process)

1. Input Byte
2. GF Inverse (\(x^{-1}\))
3. Affine Transform
4. S-Box Value

4 키 확장 (Key Expansion)

  • 128비트 초기 비밀키를 확장하여 총 44개의 워드(176바이트)로 구성된 키 스케줄을 생성합니다.
  • 규칙: 4의 배수 위치에서는 비트 회전(Rot), S-Box 치환(Sub), 라운드 상수(Rcon) 연산이 추가됩니다.

라운드 상수: Rcon (Round Constant)

각 라운드 키 생성 시 대칭성을 깨기 위해 더해지는 상수입니다.

생성 원리: 이전 값에 2를 곱합니다 (GF 상에서의 \(x\) 곱셈).
0x11b의 개입: 값이 8비트 범위를 초과할 경우 기약 다항식 0x11b와 XOR 연산을 수행합니다.

4) AddRoundKey (키 결합)

현재 State와 해당 라운드의 키(Round Key)를 비트 단위로 XOR합니다.

비밀키(Key)가 실제 데이터에 주입되는 유일한 단계이며, 복호화 시 가역성을 가집니다.

7. 복호화 (Decryption)

암호화의 역방향 프로세스로 수행되어 평문을 완벽하게 복원합니다.

  • InvSubBytes: 역 S-Box 사용
  • InvShiftRows: 오른쪽 순환 이동
  • InvMixColumns: 역행렬 곱셈

8. 결론: AES의 위대함

  • 수학적 완결성 \(GF(2^8)\)과 0x11b를 통해 오차 없는 연산 체계 구축.
  • 구조적 견고함 비선형(S-Box)과 확산(MixColumns)의 조화로 공격 방어.
  • 효율성 XOR 및 시프트 연산 위주로 고속 동작.
